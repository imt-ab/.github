# Contributing

Thank you for your interest in contributing.

This organization publishes frameworks, methods, and technical reference material related to architecture-led software development, practical AI enablement, cloud integration, cybersecurity, and real-time systems.

Contribution models may vary between repositories, so always read the repository-specific README before opening an issue or pull request.

## General principles

Please keep contributions:

- focused and limited in scope
- consistent with the architectural intent of the repository
- clearly documented where structural decisions are involved
- maintainable over time
- free from unnecessary dependencies or incidental complexity

## Before submitting changes

Please ensure that you:

1. read the repository README and any available templates
2. check whether an issue already exists
3. describe the purpose and scope of the proposed change clearly
4. keep pull requests small and reviewable

## Pull request expectations

When submitting a pull request:

- explain what has changed
- explain why the change is needed
- note any architectural implications
- update related documentation where relevant
- avoid unrelated refactoring unless clearly justified

## Style of contribution

This organization values:

- clarity over cleverness
- explicit structure over implicit assumptions
- architecture before tooling
- practical implementation over unnecessary abstraction
